2. Difference between localStorage sessionStorage cookies

~~~localStorage~~~

Answer: Once something is stored in local storage, if it is not deleted by itself, it is never deleted. In local storage (key and value) should be set according to the pair.

~~~sessionStorage~~~

Answer: If something is set in the sessionstorage, it will be deleted when the browser is loaded or when the browser is exited, there is no data in the sessionstorage.

~~~cookies~~~

Answer: Cookies are small pieces of data that are sent from a website and stored on the user's device. They can be used to remember user preferences or login information. Unlike localStorage and sessionStorage, cookies have an expiration date and can be set to expire after a certain time or when the user closes their browser. They can also be set with a path or domain restriction to limit the scope of access.


3. Event Loop

Answer: JavaScript is a single threat language. Uses an even loop to handle multitasking. Each browser has its own engine installed. Chrome browser has an engine called V8 installed and executes tasks there. And if it contains any asynchronous behavior it is supposed to transfer to the browser API. Callbacks from the browser's API are sent to the queue. The v8 engine tasks are sent to the v8 engine using an even loop in the asynchronous task in the callback queue(FIFO) after executing them sequentially. This helping hand is called an even loop.
